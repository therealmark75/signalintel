# Config Variable Classification

Authoritative reference for secrets-leakage audits on tracked config files.
Generated by AST scan on 2026-05-14. Update when tracked config files change.

**Rule:** When auditing a config-touching commit for secrets leakage, grep against this file first, not against literal patterns like `TOKEN|API_KEY|PASSWORD`. Pattern greps miss variables whose names do not contain those keywords (e.g. `ALERT_CONFIG` contains smtp credentials but matches none of those patterns).

## Maintenance

When adding a new variable to any tracked config file:
1. Add a row to this file with its classification before committing.
2. If SECRET: verify it will not be present in any tracked file with a real value (only empty-string placeholders in `.example.py` files).
3. If a new `.py` file is added under `config/` and tracked, add a section here.
